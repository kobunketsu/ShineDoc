Detailed Description

效果预设模型，所有效果统一使用相同的存储结构，这样就可以使用一个大数组存储所有效果，而不用字典的方式存储。 优点：预设之间可以定义先后作用顺序，存储结构简单。 缺点：所有效果不分类可能比较混乱，分类需要通过提取预设内的效果类型来进行。

设置参数流程 effect > effectType > name > paramDic 或者 effect > effectType > subPresetKey > subPresetDic > name > paramDic
